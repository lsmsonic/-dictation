## 1. 아키텍처 요약

- **단일 `index.html` 자기완결형**: HTML + 인라인 `<style>` + 인라인 `<script>` 한 파일.
  서버 코드/빌드 없음 → Vercel 정적 호스팅(서버리스). **이 구조를 그대로 계승.**
- **외부 의존(CDN)**:
  - `tesseract.js@5` (jsdelivr) — 클라이언트 OCR
  - `canvas-confetti@1.9.3` — 결과 축하 효과
  - Google Fonts (Fredoka, Quicksand)
- **화면 전환**: `goToScreen(screenId)` 로 단일 페이지 내 화면 토글
  → 업로드 → 검수/편집 → 설정 → 학습/퀴즈 → 결과.
- **상태**: 전역 변수(`wordsList`, `quizList`, `currentScore`, `geminiApiKey`, `preferredOcrEngine` 등) + localStorage.

## 3. Gemini 키 처리 — BYOK 유지 (수정 불필요)

레퍼런스 `runGeminiOcr` (약 2104~2180행)는 키를 클라이언트에서 직접 사용:

```js
let geminiApiKey = localStorage.getItem('gemini_api_key') || '';   // 사용자 본인이 입력→본인 기기 저장
url: `...gemini-2.5-flash:generateContent?key=${geminiApiKey}`     // 브라우저에서 직접 호출
```

**운영 모델 확정: 우리 가족 전용 + BYOK(각자 본인 키 입력).**
- 보호할 **공유 비밀(운영자 키)이 없음** → 이 클라이언트 측 구조를 **그대로 유지해도 안전**.
- 키는 본인 소유·본인 기기(localStorage)·본인이 신뢰하는 Google로 직접 전송. 우리 서버를 거치지 않음.
- 따라서 **서버리스 프록시(`/api/*`) 불필요 → 순수 정적 HTML 유지** (더 단순·더 사적).

**이식 시 유지/보강:**
- `runGeminiOcr` 호출 골격·키 입력 UI(`type=password`)·localStorage 저장 방식 **계승**.
- 프롬프트만 "한국어 받아쓰기 문장 + 띄어쓰기·문장부호 보존 추출"용으로 교체.
- JSON 정제 로직(코드펜스 제거·`[`~`]` 추출·trailing comma 제거) 그대로 재사용.
- **보강**: 키는 선택사항(미입력 시 Tesseract만), 키 삭제 버튼, 키 평문 화면/로그 노출 금지.
- **유일 리스크는 XSS**(클라이언트 키 → XSS면 키 탈취) → 모든 표시 텍스트 `escapeHtml` + CSP.

## 4. TTS — 현재 vs 필요

레퍼런스 `playWordPronunciation`:
```js
const utterance = new SpeechSynthesisUtterance(word);
utterance.lang = 'en-US';
utterance.rate = 0.85;          // ← 하드코딩, 사용자 조절 불가
window.speechSynthesis.cancel(); // (좋은 패턴: 재생 전 취소)
window.speechSynthesis.speak(utterance);
```
**받아쓰기 마스터 변경:**
- `utterance.lang = 'ko-KR'` (한국어 문장).
- `utterance.rate` 를 **전역 사용자 설정값**으로 (슬라이더/프리셋 0.5~1.5), localStorage 기억.
- 재생/다시듣기/정지 컨트롤, (선택) 문장 자동 반복·읽기 간격.
- `cancel()` 후 `speak()` 패턴은 계승.

## 5. 시험모드 채점 (주관식 완전 일치)

레퍼런스 `processAnswer(isCorrect, userText)` 구조를 재사용하되, 정답 판정을:
```js
// 받아쓰기 정답 판정(개념): 앞뒤 공백만 정규화, 그 외(부호·내부 공백·글자) 완전 일치
const norm = s => s.trim();              // 내부 공백/부호는 보존 — 받아쓰기 본질
const isCorrect = norm(userText) === norm(answerSentence);
```
- 오답 시 **글자 단위 diff** 로 틀린 글자/누락 부호/잘못된 띄어쓰기 위치 하이라이트.
- 오답 유형 분류(글자/띄어쓰기/부호)로 결과 리포트·복습 연계.
