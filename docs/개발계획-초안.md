## 1. 프로젝트 개요

촬영된 이미지를 분석하여 받아쓰기 문장을 인식하고, **학습모드**와 **시험모드**로
작동하는 **서버리스 웹앱**을 구현한다.

- **핵심 가치**: 교사·학부모가 받아쓰기 문제지/교과서 문장을 사진으로 찍으면,
  앱이 문장을 인식해 받아쓰기 학습/시험 콘텐츠로 즉시 변환.
- **정확도 요건**: **띄어쓰기·문장부호까지 정확히** 인식해야 함 (받아쓰기 채점 특성상
  맞춤법·간격·부호가 곧 정답 기준).
- **인식 전략**: 1차 **기본 OCR**, 인식 성능이 떨어질 때 **Gemini API** 기반 분석으로 폴백.
- **레퍼런스**: 기존 영어 단어 학습 앱(https://voca-study-mu.vercel.app/)의 구조·UX·코드를
  분석하여 동일 패턴으로 구현. (GitHub 소스 보유)
- **개발/배포 환경**: GitHub + Vercel.

## 2. 레퍼런스 분석 (선행 작업)

기존 `voca-study` 레퍼런스에서 재사용/이식할 요소를 먼저 정리한다.

| 분석 항목 | 확인할 내용 | 이식 방향 |
|---|---|---|
| 앱 구조 | Vanilla SPA 여부, 라우팅/화면 전환 방식 | 동일 구조 채택 |
| 학습/시험 흐름 | 단어 → 문장 단위로 어떻게 확장할지 | 문장 기반으로 재설계 |
| 데이터 저장 | localStorage 스키마, 진도·오답 저장 | 받아쓰기 세트 스키마로 확장 |
| 채점 로직 | 정답 비교 방식 | 띄어쓰기·부호 비교 로직 신설 |
| 서버리스 프록시 | API 호출 구조(있다면) | Gemini 프록시로 재활용 |
| UI/디자인 토큰 | 색·폰트·컴포넌트 | 일관성 위해 계승 |

**레퍼런스 실측 확인 (2026-06-20, 라이브 사이트 분석):**
- 흐름: 이미지 업로드 → OCR(Tesseract/Gemini) → 표로 검수·편집 → 개수 선택(5/10/20/전체) → 모드 선택 → 진행(진행률) → 결과 리포트("오답 대장"). **이 전체 흐름을 받아쓰기(문장 단위)로 계승.**
- 모드: 📖 플래시카드 학습 / ✏️ 퀴즈(**객관식 A·B·C·D**). 데이터 보관함 제공.
- TTS: "구글 입모양 발음 학습" 링크 수준, **읽기 속도 조절 없음**.

**받아쓰기 마스터의 차별점 (레퍼런스 대비 변경/추가):**
| 항목 | 레퍼런스(voca-study) | 받아쓰기 마스터 |
|---|---|---|
| 학습 단위 | 영어 **단어** | 한국어 **문장**(띄어쓰기·부호 포함) |
| 음성 | 구글 발음 링크, 속도조절 ✕ | **예시 문장 TTS + 읽기 속도 조절 (필수)** |
| 시험 | **객관식** 4지선다 | **주관식 전용**, 직접 작성 |
| 채점 | 보기 선택 일치 | **글자·띄어쓰기·문장부호 완전 일치** |

**소스 코드 정독 완료 (2026-06-20)** → 별도 문서 **[레퍼런스 코드 분석](레퍼런스-분석.md)** 참조.
핵심 결론: 단일 `index.html` 자기완결형(Tesseract.js+canvas-confetti CDN), `goToScreen` 화면전환,
이미 **주관식 입력 채점 경로(`submitSpellingAnswer`→`processAnswer`)** 존재(시험모드에 재사용),
**Gemini 호출은 키를 클라이언트에 노출(`?key=${localStorage}`)하는 안티패턴 → 서버리스 프록시로 재작성 필수**,
TTS는 `rate=0.85` 하드코딩이라 속도조절 신규 구현 필요.

## 3. 핵심 기능 요구사항

### 3.1 이미지 입력
- 카메라 촬영 / 갤러리 업로드 (모바일 우선, `<input type="file" accept="image/*" capture>`).
- 업로드 전 **클라이언트 측 리사이즈/압축** (전송량·비용·속도 절감).
- 인식 영역 미리보기 및 재촬영.

### 3.2 문장 인식 (2단계 폴백)
1. **1차 — 기본 OCR (클라이언트)**
   - 후보: Tesseract.js(한국어 `kor` 학습데이터). 완전 클라이언트 실행 → 비용 0, 키 불필요.
   - 한계: 손글씨·저해상도·복잡 배경에서 띄어쓰기/부호 정확도 저하 가능.
2. **인식 품질 판정**
   - 신뢰도 점수(평균 confidence), 빈 결과, 비정상 문자비율 등으로 **자동 폴백 트리거**.
   - 사용자 수동 "다시 분석(고정밀)" 버튼도 제공.
3. **2차 — Gemini API 폴백 (서버리스 프록시 경유)**
   - 이미지 → 텍스트(문장·띄어쓰기·부호 보존) 변환 프롬프트.
   - **BYOK**: 사용자가 입력한 본인 Gemini 키로 브라우저가 직접 호출(서버 없음, 4장 참조).
   - 키 미입력 시 이 단계 비활성 — 1차 OCR 결과만 사용.

### 3.3 인식 결과 교정 UI
- OCR/AI 결과를 **문장 리스트로 편집 가능**하게 표시 (받아쓰기 정답은 사람이 최종 확정).
- 띄어쓰기·문장부호 표시를 명확히(예: 공백 점선 시각화 옵션).
- 문장 세트 저장 → 학습/시험에서 재사용.

### 3.4 음성 읽기 (TTS) — **필수 기능**
> 받아쓰기는 "들려주고 받아쓰는" 학습이므로 **예시 문장 음성 읽기는 핵심 필수 기능**.
- **예시 문장 읽어주기**: 학습/시험 모든 화면에서 문장을 음성으로 출력 (Web Speech API `SpeechSynthesis`, 한국어 음성).
  - 재생 / 다시 듣기(반복) / 정지 컨트롤.
- **읽기 속도 조절 (필수)**: 느리게 ↔ 빠르게 단계 조절 (`utterance.rate`, 예: 0.5 / 0.75 / 1.0 / 1.25 / 1.5 또는 슬라이더).
  - 받아쓰기 받아적기 편의를 위해 **저속(또박또박) 읽기** 프리셋 제공.
  - 마지막 선택 속도 localStorage에 기억.
- (선택) 문장 단위 자동 반복 횟수·읽기 간격(딜레이) 설정 — 실제 받아쓰기 받아적기 시간 확보.
- 폴백: 기기/브라우저가 한국어 TTS 미지원 시 안내 및 텍스트 표시 유지.

### 3.5 학습모드
- 예시 문장을 **듣고(TTS, 3.4)** 따라 쓰기, 정답 즉시 확인.
- 문장 표시 후 음성 반복 청취 가능.
- 틀린 부분(글자/띄어쓰기/부호) 하이라이트.
- 오답 문장 반복 학습.

### 3.6 시험모드 — **주관식 전용**
> 레퍼런스(voca-study)의 시험은 **객관식**이지만, 받아쓰기 마스터는 **주관식만** 지원.
- **출제**: TTS(3.4)로 문장을 읽어주고(속도 조절 가능), 사용자가 **직접 입력/작성**.
  - 객관식·보기 선택지 **없음**.
- **채점 — 문장부호까지 정확히 동일해야 정답**:
  - 정답 문장과 **글자·띄어쓰기·문장부호가 완전히 일치**해야 정답 처리 (받아쓰기 본질).
  - 비교 시 앞뒤 공백 trim 정도만 정규화, 그 외 차이(부호 누락/공백 차이/오타)는 모두 오답.
  - 틀린 위치를 글자 단위 diff로 하이라이트(틀린 글자/누락 부호/잘못된 띄어쓰기 표시).
- 결과 리포트: 점수, 틀린 문장, 오답 유형(글자/띄어쓰기/부호) 분류, 오답 복습 연계.

### 3.7 데이터/진도
- localStorage 기반 세트·진도·오답 저장(서버리스, 무계정 우선).
- 세트 내보내기/가져오기(JSON) — 기기 간 공유.

## 4. 아키텍처 & 보안 (시큐어코딩 · OWASP Top 10)

> **운영 모델 (확정): 우리 가족 전용 + BYOK(Bring Your Own Key).**
> 운영자가 공유하는 키가 **없으므로**, 사용자가 **본인의 Gemini 키를 직접 입력**해
> 성능을 개선하는 방식. 사용량 제한·디바이스 제한·서버 프록시 **불필요**.
> → **순수 정적 "서버리스 HTML"** (Vercel 정적 호스팅, `/api` 서버함수 없음).

```
[브라우저: HTML/JS]  ── 단일 index.html, 서버 코드 없음
  ├─ 1차 OCR: Tesseract.js (클라이언트, 키 불필요 — 기본 동작)
  └─ 2차(성능개선): 사용자가 입력한 본인 Gemini 키로 브라우저가 직접 호출
        key: localStorage('gemini_api_key')  ── 본인 기기에만 저장
                     │ (HTTPS, 사용자→Google 직접)
              [Google Gemini API]
```

> **위협 모델 메모**: 키는 **각 사용자 본인 소유**이고 **본인 기기에만** 저장되며
> **본인이 신뢰하는 Google로 직접** 전송된다. 보호해야 할 "공유 비밀"이 없으므로
> 레퍼런스(voca-study)의 localStorage 키 보관 방식을 **그대로 유지**해도 안전.
> (운영자 공유 키였다면 서버 프록시가 필수였으나, 이 모델에서는 불필요.)

### 4.1 BYOK 모델에서 지킬 것 (시큐어코딩)
- **키는 본인 기기 localStorage에만** 저장. 우리 서버/외부로 절대 전송·로깅하지 않음.
- 키 입력란은 `type="password"`, 화면/로그/이력 어디에도 키 평문 노출 금지.
- 키는 **선택사항** — 미입력 시 Tesseract.js로 정상 동작(키 없어도 앱 사용 가능).
- 키 삭제(초기화) 버튼 제공.

### 4.2 OWASP Top 10 대응 (정적·BYOK 맥락)

| 항목 | 위험 | 대응 |
|---|---|---|
| A02 암호화 실패 | 키 유출 | 키 localStorage 한정·외부 전송 금지, 전구간 HTTPS, `type=password` |
| A03 인젝션(XSS) | 인식/AI 텍스트로 XSS → localStorage 키 탈취 | **모든 표시 텍스트 escapeHtml**(레퍼런스 함수 계승), CSP 헤더 |
| A05 보안 설정 오류 | 보안 헤더 미설정 | Vercel `vercel.json`에 CSP·X-Content-Type-Options 등 |
| A06 취약 의존성 | CDN 변조 | CDN 버전 핀 고정(`tesseract.js@5`) + **SRI 무결성 해시** |
| A08 무결성 실패 | 외부 스크립트 | 신뢰 CDN만, SRI 적용 |
| A10 SSRF | — | 외부 호출은 Google Gemini로 한정(고정 URL) |

> XSS가 이 앱의 **유일·최대 보안 포인트**: 클라이언트 키 모델에선 XSS가 곧 키 탈취.
> OCR/AI가 만든 텍스트는 **신뢰하지 말고** 반드시 escape 후 표시.

### 4.3 개인정보
- 촬영 이미지는 인식에만 쓰고 **저장하지 않음**(처리 후 폐기). 서버가 없으므로 서버 보관 자체가 없음.
- 가족 전용·무계정이라 외부 수집 없음. Gemini 사용 시 이미지가 Google로 전송됨을 사용자에게 고지.

## 5. 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 프론트 | Vanilla HTML/CSS/JS (레퍼런스 계승) | 빌드리스 또는 경량 번들 |
| OCR(1차) | Tesseract.js (kor) | 클라이언트 실행, 무료 |
| AI(2차) | Google Gemini API | **BYOK** — 사용자 본인 키로 브라우저 직접 호출 |
| 백엔드 | **없음 (순수 정적)** | BYOK라 프록시 불필요, Vercel 정적 호스팅 |
| 저장 | localStorage | 무계정, 키·세트·진도 본인 기기 보관 |
| TTS | Web Speech API (`SpeechSynthesis`) | 예시 문장 읽기 + **속도 조절(필수)**, 한국어 음성 |
| 배포 | GitHub → Vercel (CI/CD) | 프리뷰·프로덕션 |

## 6. 개발 단계 (Phase)

- **P0 레퍼런스 분석** ✅: GitHub 소스(`index.html`) 정독·함수 매핑 완료 → [레퍼런스 코드 분석](레퍼런스-분석.md).
- **P1 스캐폴딩**: 프로젝트 구조, 라우팅/화면, 디자인 토큰 이식.
- **P2 이미지 입력 + 1차 OCR**: 촬영/업로드, 압축, Tesseract.js 인식, 결과 편집 UI.
- **P3 Gemini 폴백(BYOK)**: 사용자 키 입력 UI(선택·password·삭제), 본인 키로 직접 호출, 문장 추출 프롬프트, 품질 판정 자동 폴백. (레퍼런스 `runGeminiOcr` 프롬프트만 교체해 재사용)
- **P4 음성 읽기(TTS) + 학습모드**: 예시 문장 읽어주기·**속도 조절(필수)**·반복, 따라 쓰기·즉시 채점·오답 반복.
- **P5 시험모드(주관식 전용)**: TTS 출제, 직접 작성, **문장부호까지 완전 일치 채점**, diff 하이라이트, 결과 리포트.
- **P6 보안 점검**: XSS(escapeHtml)·CSP/보안헤더(`vercel.json`)·CDN SRI·키 비노출 확인, 개인정보 고지.
- **P7 배포**: GitHub(lsmsonic/-dictation) → Vercel 정적 배포(서버 함수 없음), 프리뷰·프로덕션.
